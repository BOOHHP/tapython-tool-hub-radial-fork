# 2026-05-21 — 工具详情交互、收藏筛选与卡片排版优化

## 背景

本轮围绕 Tool Hub 前台体验做连续优化：工具详情页操作按钮位置、版本对比返回路径、收藏按钮的真实功能，以及主页收藏筛选和工具卡片标题区排版。

## 详情页操作区

工具详情页原先将“收藏 / 版本对比 / Markdown / 下载 ZIP”放在标题区域右侧，窄屏或信息较多时会挤压主内容。

调整后：

- 操作按钮移动到来源信息下方的横向操作栏。
- 桌面端保持一行排列。
- 移动端按钮自动铺满并换行，避免溢出。

## 版本对比返回路径

版本对比页已补充明确的返回入口：

- 页面顶部增加“返回工具详情”按钮。
- 返回区展示当前工具名和版本对比上下文。
- 避免从详情页进入对比后缺少导航出口。

## 收藏功能

收藏按钮从纯展示按钮改为本机浏览器持久化功能：

- 使用 `localStorage` 保存收藏工具 slug。
- 点击后在“收藏 / 已收藏”之间切换。
- 已收藏状态使用实心星标与高亮样式。
- 刷新页面后收藏状态仍保留。
- 再次点击可取消收藏。

当前收藏实现不依赖后端账户体系，适合现阶段本地/内网工具库使用。未来如果需要多设备同步，可扩展为登录用户 + 后端收藏表。

## 主页收藏筛选

主页左侧工具目录新增“我的工具 / 收藏工具”栏目：

- 点击“收藏工具”只展示已收藏工具。
- 收藏筛选可与搜索、分类、风险等级、审核状态组合使用。
- 清空筛选会恢复全部工具。
- 无收藏或无匹配结果时显示对应空状态。
- 工具卡片显示“已收藏”标识，方便扫描。

## 工具卡片排版修复

收藏后，卡片右上角的“已收藏 + approved”标签会占据 Ant Card `extra` 区域，导致工具名称被挤压并过早省略。

本次将卡片头部改为两行结构：

1. 第一行：工具头像、工具名称、作者，占完整标题宽度。
2. 第二行：收藏标识和审核状态标签，独立横排并可换行。

这样收藏状态变化不会再压缩工具名，标题区在不同工具名长度下更稳定。
